whatsweb

A great-DX WhatsApp Web SDK with no browser.

No Puppeteer, no headless Chromium. It speaks the native Multi-Device protocol directly over a WebSocket, wrapped in a fully-typed, bot-friendly API.

TypeScript · ESM Node ≥ 20 Built on Baileys 7 MIT
📚 API Docs ★ GitHub Quick start

Features

🚀

No browser

A fraction of the RAM and startup time of Puppeteer-based clients.

🤖

Command router

command, hears, middleware and a rich Context.

🔀

Two styles

Event callbacks or a linear for await async iterator.

📎

Media anywhere

Path, http(s)/S3 URL, Buffer or stream — with auto type detection.

💬

Rich messages

Polls, mentions, stickers, contacts, edit / delete / forward.

👥

Groups

Metadata, participants, invite links, create / join / leave.

👤

Presence & profile

Presence, profile pictures, status and block list.

🛟

Resilient

Auto-reconnect with backoff and optional send rate-limiting.

🆔

LID-aware

Supports WhatsApp's anonymous identity system out of the box.

Quick start

$ npm install @pablofdezruiz/whatsweb

Requires Node ≥ 20 and an ESM project ("type": "module").

import { createClient } from '@pablofdezruiz/whatsweb';

const wa = createClient({ session: 'my-bot' });

wa.on('ready', (me) => console.log('Connected:', me.number));
wa.command('ping', (ctx) => ctx.reply('pong 🏓'));
wa.hears(/hello/i, (ctx) => ctx.react('👋'));

await wa.start(); // scan the QR from WhatsApp → Linked devices

Do more, with less code

Send from any link (S3, CDNs)

// auto-detects image / video / audio / document
await wa.sendFromLink(to,
  'https://bucket.s3.amazonaws.com/report.pdf');

Conversation flows

wa.command('order', async (ctx) => {
  await ctx.reply('What would you like?');
  const a = await ctx.awaitReply();
  await ctx.reply(`Got it: ${a.text}`);
});
